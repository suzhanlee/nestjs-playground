# 경계값 분석 가이드

## 경계값 분석의 원칙

경계값 분석(Boundary Value Analysis)은 입력값의 경계에서 오류가 발생할 확률이 높다는 원칙에 기반합니다.

### 기본 경계값 집합

| 데이터 타입 | 경계값 |
|-------------|--------|
| 숫자 (정수) | MIN-1, MIN, MIN+1, MAX-1, MAX, MAX+1 |
| 숫자 (실수) | MIN-ε, MIN, MIN+ε, MAX-ε, MAX, MAX+ε |
| 문자열 | 빈 문자열, 1글자, 최대길이-1, 최대길이, 최대길이+1 |
| 배열/리스트 | 빈 배열, 1개 요소, 최대-1, 최대, 최대+1 |
| 날짜 | 이전, 당일, 다음날 |
| 순서형 | 첫 번째, 중간, 마지막 |

## 문자열 경계값

### 일반 문자열

| 경계값 | 테스트 케이스 | 예상 동작 |
|--------|---------------|-----------|
| 빈 문자열 | `""` | 400 Bad Request (필수인 경우) |
| 공백만 | `"   "` | trim 후 유효성 검증 |
| 1글자 | `"a"` | 최소 길이 허용 시 정상 |
| 최소길이-1 | (minLength - 1글자) | 400 Bad Request |
| 최소길이 | (minLength 글자) | 정상 |
| 최소길이+1 | (minLength + 1글자) | 정상 |
| 최대길이-1 | (maxLength - 1글자) | 정상 |
| 최대길이 | (maxLength 글자) | 정상 |
| 최대길이+1 | (maxLength + 1글자) | 400 Bad Request |
| null | `null` | 400 Bad Request (필수인 경우) |
| undefined | `undefined` | 400 Bad Request (필수인 경우) |

### 특수 문자열 케이스

| 케이스 | 예시 | 검증 포인트 |
|--------|------|-------------|
| HTML 태그 포함 | `"<script>alert('xss')</script>"` | XSS 방지, 에스케이프 |
| SQL 인젝션 | `"'; DROP TABLE--"` | SQL 인젝션 방지 |
| 공백이 많음 | `"a   b"` | 공백 처리 정책 |
| 유니코드 | `"가나다"` | 인코딩 처리 |
| 이모지 | `"😀"` | UTF-8 지원 |

### 이메일 경계값

| 케이스 | 예시 | 예상 동작 |
|--------|------|-----------|
| 정상 | `"user@example.com"` | 정상 |
| 빈 문자열 | `""` | 400 |
| @ 없음 | `"userexample.com"` | 400 |
| 도메인 없음 | `"user@"` | 400 |
| 사용자명 없음 | `"@example.com"` | 400 |
| @ 2개 | `"user@@example.com"` | 400 |

## 숫자 경계값

### 일반 숫자

| 경계값 | 테스트 케이스 | 예상 동작 |
|--------|---------------|-----------|
| 음수 | `-1` | 제한 시 400 |
| 0 | `0` | 0 허용 여부 확인 |
| 1 | `1` | 최소값 허용 시 정상 |
| 최소값-1 | `min - 1` | 400 Bad Request |
| 최소값 | `min` | 정상 |
| 최소값+1 | `min + 1` | 정상 |
| 최대값-1 | `max - 1` | 정상 |
| 최대값 | `max` | 정상 |
| 최대값+1 | `max + 1` | 400 Bad Request |
| null | `null` | 필수 여부에 따라 |
| 실수로 정수 자리 | `"10"` (문자열) | 타입 변환 또는 400 |

### 가격/금액 경계값

| 경계값 | 테스트 케이스 | 예상 동작 |
|--------|---------------|-----------|
| 0원 | `0` | 0원 허용 여부 |
| 1원 | `1` | 최소 금액 |
| 음수 | `-1000` | 400 (금액은 음수 불가) |
| 소수점 | `"1000.5"` | 정수만 허용 시 400 |
| 매우 큰 값 | `Number.MAX_SAFE_INTEGER` | 오버플로우 확인 |

### 재고 수량 경계값

| 경계값 | 테스트 케이스 | 예상 동작 |
|--------|---------------|-----------|
| 0개 | `0` | 품절 상태 |
| 1개 | `1` | 최소 재고 |
| 최대 재고-1 | `maxStock - 1` | 정상 |
| 최대 재고 | `maxStock` | 정상 |
| 최대 재고+1 | `maxStock + 1` | 400 |
| 감소 시 0 미만 | `stock - quantity < 0` | 400 (재고 부족) |

## 배열/리스트 경계값

### 일반 배열

| 경계값 | 테스트 케이스 | 예상 동작 |
|--------|---------------|-----------|
| 빈 배열 | `[]` | 정상 (빈 목록) |
| 1개 요소 | `[item]` | 정상 |
| 최소-1개 | `minItems - 1` | 400 |
| 최소 개수 | `[...minItems]` | 정상 |
| 최소+1개 | `[...minItems + 1]` | 정상 |
| 최대-1개 | `[...maxItems - 1]` | 정상 |
| 최대 개수 | `[...maxItems]` | 정상 |
| 최대+1개 | `[...maxItems + 1]` | 400 |
| null | `null` | 필수 여부에 따라 |
| undefined | `undefined` | 필수 여부에 따라 |

## 날짜/시간 경계값

### 일반 날짜

| 경계값 | 테스트 케이스 | 예상 동작 |
|--------|---------------|-----------|
| 과거 날짜 | `"2020-01-01"` | 과거 날짜 허용 여부 |
| 오늘 | 오늘 날짜 | 정상 |
| 미래 날짜 | `"2099-12-31"` | 미래 날짜 허용 여부 |
| 형식 오류 | `"01-01-2024"` | 400 (잘못된 형식) |
| 윤일 | `"2024-02-29"` | 윤년 처리 |
| 존재하지 않는 날짜 | `"2024-02-30"` | 400 |

### 시작일/종료일 쌍

| 케이스 | 테스트 케이스 | 예상 동작 |
|--------|---------------|-----------|
| 정상 범위 | startDate < endDate | 정상 |
| 같은 날 | startDate == endDate | 정상 (허용 시) |
| 역전 | startDate > endDate | 400 |
| 겹침 | 범위가 겹치는지 | 비즈니스 규칙에 따름 |

## 파일 업로드 경계값

| 케이스 | 테스트 케이스 | 예상 동작 |
|--------|---------------|-----------|
| 빈 파일 | 0 bytes | 400 (빈 파일 불가) |
| 최대 크기-1 | maxSize - 1 | 정상 |
| 최대 크기 | maxSize | 정상 |
| 최대 크기+1 | maxSize + 1 | 400 |
| 허용된 확장자 | `.jpg`, `.png` | 정상 |
| 허용되지 않은 확장자 | `.exe` | 400 |
| MIME 불일치 | `.jpg`지만 내용은 다른 파일 | 400 |

## 페이지네이션 경계값

|--------|---------------|-----------|

| 마지막 페이지 | 마지막 페이지 번호 | 정상 |

| 최대 페이지 크기 초과 | `pageSize=1000` | 400 |
